Log Rotation & Cleanup // 日誌輪替與清理

Log entries are grouped into one file per month and can be reviewed in the Logs tab. Old files are removed automatically according to the rules below. // 日誌記錄按月份分組為一個檔案，可在「日誌」頁面檢閱。舊檔案將依照以下規則自動刪除。

Keep Logs For (months) // 日誌保留時間（月）
Monthly log files older than this are deleted. Set 0 to keep logs forever. // 超過此時間的每月日誌檔案將被刪除。設為 0 表示永久保留。
Max Total Log Size (MB) // 日誌總大小上限（MB）
When the log folder grows beyond this size, the oldest files are purged first. Set 0 for no size limit. The current month's file is never deleted. // 當日誌資料夾超過此大小時，將優先清除最舊的檔案。設為 0 表示不限制大小。本月的日誌檔案永遠不會被刪除。
